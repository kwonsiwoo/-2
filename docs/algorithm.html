찐막차 경로 추출 알고리즘

odsayService.ts · tmapService.ts · transitScheduleService.ts 기반 | 작성일: 2026-04-22

① 전체 파이프라인

1
지오코딩 — 주소 → 좌표 변환
TMAP 도로명주소 → OSM Nominatim → TMAP POI 순으로 시도. 주소를 최대 7가지 변형 후보로 확장해 각각 시도.
2
ODsay API 원시 경로 수신
출발·도착 좌표를 서버 프록시(/api/odsay)에 전달. 출발 시각 지정 시 SearchDate·SearchTime 파라미터 추가.
3
경로 유효성 필터링 출발 시각 지정 시만 작동
총 소요시간 240분 초과 제거 / 유휴 시간 30분 초과 제거 / TAGO API로 실제 운행 여부 확인.
4
택시 제외 모드 분기
excludeTaxi=true이면 MCDM 엔진 스킵 → 순수 대중교통 경로 3개 즉시 반환.
5
MCDM 하이브리드 경로 생성
시간대 감지 → 3가지 전략(time-saving / cost-saving / balanced)으로 각각 경로 생성. 경로당 택시 1회 삽입.
6
클라이언트 사이드 필터 적용
최대 택시비 / 최대 도보 시간 / 최대 환승 횟수 조건으로 화면 출력 직전 추가 필터링.

③ 경로 유효성 필터링

출발 시각 지정 시에만 작동 3가지 조건을 모두 통과해야 유효한 경로로 사용됨.

조건 1 — 총 소요시간 제한

totalTime > 240분  →  제외

조건 2 — 유휴 시간 과다 제거

totalTime - Σ(sectionTime) > 30분  →  제외

대기 시간이 비정상적으로 많은 경로 제거

조건 3 — 실제 운행 가능 여부 (transitScheduleService)

세그먼트운행 가능 시간불가 시간판단 방식
도보항상-즉시 통과
지하철 (type=1)05:30 ~ 00:3001:00 ~ 05:29시간대 직접 비교
일반버스 (type=2)04:30 ~ 24:0001:00 ~ 04:29시간대 직접 비교
심야버스 N버스 (routetp=16)23:30 ~ 05:30-TAGO API 노선유형 조회

심야버스 판별 순서

  1. 버스 번호가 N으로 시작하면 즉시 심야버스 판정
  2. TAGO BusRouteInfoInqireService API로 routetp 조회 → 16이면 심야버스
  3. API 실패 시 → N\d 패턴 기반 폴백
  4. 조회 결과는 24시간 메모리 캐시 적용

전략 × ODsay 경로 매핑

전략사용 ODsay 경로의미
time-saving0번 (최상위)ODsay가 가장 빠르다고 추천한 경로 기반
cost-saving0번 (최상위)동일 경로, 택시 삽입 전략만 다름
balanced1번 (차선)두 번째 경로 기반, 다양성 확보

택시 삽입 분기 — 경로당 택시 1회

택시 후보 조건: 도보 구간 중 max(사용자 설정 도보 임계값, 10분) 초과인 것만.
기본 임계값 = 20분 / 최소 하드 하한 = 10분 (사용자가 10분 이하로 설정해도 10분 미만 도보는 택시 불가)

Case A — 도보 구간을 택시로 대체 (후보 존재 시)

모드time-savingcost-savingbalanced
야간 마지막 도보
(막차 의존 최소화)		 가장 짧은 도보
(최소 비용)		 포지션 가중 / 택시비
점수 최대
(마지막=3.0×, 그 외=1.5×)
주간 효율점수 × 위치 가중치 최대
(첫번째=1.3×, 마지막=1.2×, 그 외=1.0×)		 마지막 도보 순수 효율점수 최대

택시 요금 계산 (도보 대체)

도보거리(m) = (도보분 / 60) × 4,000m
택시요금    = 4,800원
              + ceil((거리m - 1,600) / 131) × 100원
분단축      = round(도보분 × (1 - 4/25))  ≈ 84%
택시시간    = max(1, 도보분 - 분단축)

효율점수 (timeValueScore)

효율점수 = 분단축 / 택시비 × 1,000
(단위: 분/천원, 클수록 가성비 좋음)

Case B — 환승 지점에서 목적지까지 택시 (도보 후보 없을 때)

각 대중교통 종점을 후보로 평가:

  • 목적지까지 Haversine 직선 거리 계산
  • 0.3km 미만 지점 제외 (너무 가까움)
  • 점수 = (남은경로시간 - 택시시간) / 택시비 × 1000
  • 시간 절약이 없으면 점수 = -거리(km) (거리 기준 정렬로 전환)
모드time-savingcost-savingbalanced
야간 마지막 환승 지점 목적지와 가장 가까운 지점 점수 최대
주간 점수 최대 목적지와 가장 가까운 지점 점수 최대

택시 요금 계산 (환승 지점)

직선거리(km) = Haversine(환승지점, 목적지)
택시요금     = 4,800원 + ceil((거리m - 1,600) / 131) × 100원
택시시간     = max(2, round(거리km / 25 × 60))분

⑤ 클라이언트 사이드 필터

API 재호출 없이 이미 수신한 경로에서 화면 출력 직전 추가 필터링.

필터조건API 재호출
최대 택시비route.taxiWalkCost > filterMaxTaxi → 제외No
최대 도보 시간잔여 도보 합계 > filterMaxWalk → 제외Yes (임계값 변경 시)
최대 환승 횟수route.transferCount > filterMaxTransfer → 제외No
출발 시각 지정해당 시각으로 ODsay 재조회Yes
택시 제외 전환buildPureRoute로 ODsay 재조회Yes
